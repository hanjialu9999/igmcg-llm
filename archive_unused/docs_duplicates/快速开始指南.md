| 项目 | CPU | GPU |

```

```

## ⚙️ 配置文件详解

### `config/config.yaml` - 训练配置

```yaml
model:
  embedding_dim: 512      # 词嵌入维数（越大越强但更慢）
  num_heads: 8            # 注意力头数（通常8或16）
  num_layers: 6           # Transformer层数（越多越复杂但可能过拟合）
  hidden_dim: 1024        # 前馈网络隐藏层维数

training:
  batch_size: 128         # GPU优化值；12GB显存用64，24GB用256
  epochs: 200             # 总训练轮数（通常150-300）
  learning_rate: 0.0005   # 基础学习率（带warmup机制）
  weight_decay: 0.0001    # L2正则化强度（防过拟合）
  gradient_clip: 1.0      # 梯度裁剪（防梯度爆炸）
  warmup_steps: 30        # 预热步数（学习率从0线性增长）
  early_stop_patience: 10 # 多少个epoch验证loss无改进就停止

data:
  train_file: "data/train_data_final.txt"  # 训练数据
  max_seq_length: 64      # 最大序列长度（越大内存越多）
  num_workers: 4          # 并行加载数据的进程数

device: "cuda"            # 使用GPU；如无GPU改为 "cpu"
```

### `config/dialogue_params.json` - 生成参数

```json
{
  "temperature": 0.68,        // 值越小越确定，越大越多样
  "top_k": 42,                // 每次只考虑top-K个概率最高的词
  "repetition_penalty": 2.0,  // 防止重复，值越大越防重复
  "max_length": 15            // 最多生成多少个词
}
```

**参数建议：**
- 保守模式：temp=0.3, top_k=20, penalty=3.0 (确定但重复)
- 平衡模式：temp=0.68, top_k=42, penalty=2.0 (推荐👍)
- 创意模式：temp=1.0, top_k=60, penalty=1.5 (多样但可能乱)

## ✅ 训练流程详解

### 自动步骤
1. **加载数据** → 从 `train_data_final.txt` 读取 QA对
2. **分割集合** → 80% 训练，20% 验证
3. **初始化模型** → Transformer 6层，**在GPU上**
4. **训练循环** (每个epoch):
   - Warmup阶段 (30步): 学习率从0增长到目标值
   - 训练阶段: 计算loss，反向传播，更新权重
   - 验证阶段: 在验证集上评估
   - **自动保存checkpoint** (保留最后5个epoch)
5. **Early Stopping**: 如果验证loss连续10个epoch无改进，停止训练
6. **保存最终模型** → `checkpoints/final_model.pt`

### 检查点管理
```
checkpoints/
├── epoch_1.pt        ← 第1轮
├── epoch_2.pt        ← 第2轮
├── ...
├── epoch_195.pt      ← 最后5个checkpoint保留
├── epoch_196.pt
├── epoch_197.pt
├── epoch_198.pt
├── epoch_199.pt
└── final_model.pt    ← 🎯 最终模型 (这个自动加载)
```

**中断恢复**: 如果训练中断，重新运行会从最后一个 epoch 自动继续！

## 🎨 参数调优指南

### When to adjust 什么时候调参数?

| 问题 | 原因 | 解决 |
|------|------|------|
| Loss 下降缓慢 | 学习率太小 | 增加 learning_rate 到 0.001 |
| Loss 振荡/不稳定 | 学习率太大 | 减小 learning_rate 到 0.0001 |
| 过拟合 (训练loss低，验证loss高) | 模型太强 | 增加 weight_decay 到 0.001 |
| 欠拟合 (两个loss都高) | 数据太少或模型太弱 | 增加 epochs 到 300，或增加 num_layers |
| GPU显存溢出 | batch_size太大 | 改为 64 或 32 |
| 训练太慢 | batch_size太小 | 改为 256 (如果显存允许) |

### 生成质量调优

在 `dialogue_interactive.py` 中输入 `config` 实时调整：


## 🔧 常见问题

### Q: GPU显存不足怎么办？

**A:** 在 `config/config.yaml` 中改小 batch_size:

```yaml
# 原来
training:
  batch_size: 128

# 改为
training:
  batch_size: 64  # 或 32 如果还是溢出
```

**内存参考：**
- RTX 3060 (12GB): batch_size = 64
- RTX 3080 (10GB): batch_size = 96
- RTX 3090 (24GB): batch_size = 256
- A100 (80GB): batch_size = 512

### Q: 训练突然停止了怎么办？

**A:** 有几种可能：
1. **Early stopping 触发** → 验证loss 10个epoch无改进自动停止
2. **GPU出错** → 查看 `training.log` 最后几行错误信息
3. **系统关闭** → 重新运行会自动从上次的 epoch 恢复

```bash
# 查看日志最后错误
tail -20 training.log      # 或 30 行

# 重新启动 (自动恢复)
python scripts/train.py --config config/config.yaml
```

### Q: 如何知道训练完成了？

**A:** 几种方式：

1. **检查日志**
   ```bash
   tail training.log
   # 会看到: Epoch 150/200 完成 或 Training completed!
   ```

2. **检查最终模型文件**
   ```bash
   ls -lh checkpoints/final_model.pt
   # 如果文件时间很近，说明刚完成
   ```

3. **启动对话系统**
   ```bash
   python dialogue_interactive.py
   # 然后测试模型是否正常工作
   ```

### Q: 新训练的模型会自动加载吗？

**A:** 是的！`dialogue_interactive.py` 和其他脚本自动加载 `checkpoints/final_model.pt`。

如果想用特定的checkpoint版本，修改加载代码：
```python
# 在 dialogue_interactive.py 中找到这行
model = torch.load('checkpoints/final_model.pt')  # ← 改这里
model = torch.load('checkpoints/epoch_150.pt')    # ← 用第150个epoch
```

### Q: 显存溢出错误是什么？

```
RuntimeError: CUDA out of memory. 
Tried to allocate X.XXGiB but available memory is only Y.YYGiB
```

**解决方法:**
1. **减小 batch_size** (改为 64, 32, 16)
2. **减小 max_seq_length** (改为 48 或 32)
3. **关闭其他程序** 释放显存
4. **减少 num_workers** (改为 2 或 1)

```yaml
# 编辑 config/config.yaml
training:
  batch_size: 32      # ↑ 改这个
data:
  max_seq_length: 48  # ↑ 或这个
  num_workers: 2      # ↑ 或这个
```

## 🎓 深入理解

### 为什么GPU这么快?

| 操作 | CPU | GPU |
|------|-----|-----|
| 矩阵乘法 (1000x1000) | 逐个计算 | **并行计算** |
| 单位精度 | 64位float | 32位float |
| 吞吐量 | ~100 GFLOPS | ~10,000 GFLOPS (100倍!) |

GPU的 **数千个核心** 并行处理大量计算，非常适合深度学习！

### 混合精度为什么安全?

- 梯度计算: float16 (快)
- 权重更新: float32 (精)
- GradScaler: 防止梯度下溢

→ 结果: **训练快2-4倍，精度完全相同**

### Early Stopping 如何工作?

```
Epoch 1-40:    验证loss 持续下降 ✓
Epoch 41-50:   验证loss 停止改进 ⚠️
Epoch 51:      连续10个epoch无改进 → 停止!
```

防止过拟合，自动找到最优点。
